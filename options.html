Salesforce Connected App configuration checklist

If any test step fails, these are the exact Salesforce settings to check. Go to Setup → App Manager, find your connected app, click Edit.

1. Callback URL must include

https://<extension-id>.chromiumapp.org/sfdc — this extension's specific redirect URL is shown above once the page loads. The connected app accepts multiple URLs; add this one alongside any existing ones.

2. Selected OAuth Scopes
  • Manage user data via APIs (api) — required
  • Access the identity URL service (id, profile, email, address, phone) — required
  • Perform requests at any time (refresh_token, offline_access) — required for refresh-token support
3. Web App Settings
  • Require Secret for Web Server Flow — UNCHECKED (we have no client secret; browser extensions are public clients)
  • Require Secret for Refresh Token Flow — UNCHECKED
  • Require Proof Key for Code Exchange (PKCE) Extension — recommended CHECKED
4. OAuth Policies (Manage Connected Apps → your app → Edit Policies)
  • Permitted Users: All users may self-authorize (or pre-approve specific profiles)
  • IP Relaxation: Relax IP restrictions (or whitelist your user IPs)
  • Refresh Token Policy: Refresh token is valid until revoked

After editing any of the above, Salesforce can take up to 10 minutes to propagate changes before the token endpoint reflects them.
